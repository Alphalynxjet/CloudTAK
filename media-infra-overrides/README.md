# SRT per-lease encryption + media-infra overrides

This directory holds **takserver.host** patches to the upstream
`ghcr.io/dfpc-coe/media-infra` image. media-infra runs its TypeScript at runtime
(`npx tsx index.ts`), so instead of forking and rebuilding the whole image we
**bind-mount** a few patched files over it in `docker-compose.yml`. This keeps us
on the upstream image tag and easy to upgrade.

## What this feature does

When **Read/Write Security** is enabled on a video lease, CloudTAK now also
generates a random **SRT passphrase** (alongside the read/write usernames &
passwords). media-infra pushes that passphrase into the MediaMTX path config as
`srtReadPassphrase`/`srtPublishPassphrase`, so the SRT stream is **AES-encrypted**
end to end. The passphrase is shown in the lease modal (dedicated field + inline
in the SRT URL) and is nulled when Read/Write Security is turned off.

> ⚠️ Encryption is **SRT-only**. RTMP is always cleartext and HLS relies on HTTPS.
> For confidentiality on other protocols, keep using the VPN tunnel.

## Files changed for this feature

Version bumped to **`TG.005.0`** (`api/package.json`, `api/web/package.json`) —
shown in the UI ServerStatus, so operators can confirm the build is deployed.

### CloudTAK API (`api/`)
| File | Change |
|------|--------|
| `api/package.json`, `api/web/package.json` | Version bump `TG.004.0` → `TG.005.0`. |
| `api/lib/schema.ts` | New `srt_pass` column on `video_lease`. |
| `api/migrations/0180_srt_passphrase.sql` | `ALTER TABLE ... ADD COLUMN srt_pass` (generated by drizzle-kit). |
| `api/lib/control/video-service.ts` | `updateSecure()` generates/rotates/nulls `srt_pass` with the secure creds; `protocols()` appends `&passphrase=…` to the secure SRT URL **and fixes a pre-existing `}}` double-brace typo** in the read/write streamid. |
| `api/derived-types.d.ts` | Regenerated types include `srt_pass` (also patchable via `npm run typegen`). |

`VideoLeaseResponse` is a `createSelectSchema` over the whole table, so `srt_pass`
is exposed automatically on `GET /api/video/lease` (the endpoint media-infra
polls) and `GET /api/video/lease/:lease` — no route changes needed.

### CloudTAK Web (`api/web/`)
| File | Change |

### media-infra overrides (this directory — bind-mounted)
| File | Change |
|------|--------|

| `lib/payload.ts` | `createPayload()` sets both passphrases on the MediaMTX path when the lease has one. |

### Compose
| File | Change |
|------|--------|
| `docker-compose.yml` | Bind-mounts the three override files into the `media` container **and** fixes the SRT port to `:8890/udp` (SRT is UDP — the previous TCP-only mapping meant no SRT packet ever reached MediaMTX). |

## Maintenance notes
- These overrides are pinned to the file layout of the media-infra tag in
  `docker-compose.yml` (currently `v9.1.1`). **When bumping that tag, re-diff these
  three files against the new upstream versions** before trusting the mount.
- Mounts are read-only (`:ro`).
- To refresh the API types after the schema change:
  `cd api && npm run typegen` (needs the API running).
